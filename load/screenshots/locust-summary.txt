NutriSmart — Locust load test summary (headless run, text capture)
==================================================================
Tool: Locust 2.44.1   |   Scenario: load/locustfile.py
Load: 20 users, spawn-rate 5/s, run-time 60s
Target: local run of the production Docker image (nutrismart-backend) over HTTP.

  Why local-image instead of the live HTTPS URL: Locust's gevent-patched SSL
  fails on this Windows + Python 3.13 build (every remote HTTPS request returns
  "status 0", although plain curl/PowerShell requests to the same URL return
  200). Load-testing a local run of the exact production image over HTTP avoids
  the client-side SSL defect and measures the real app stack (gunicorn + Flask +
  OpenRouter + Supabase). The /health latency gate is independently verified on
  the live Railway URL below.

Note on "fails": every request returned HTTP 200. The failures Locust reports
here are our own catch_response latency SLAs (chat > 2s, health > 500ms), NOT
transport/HTTP errors. True HTTP error rate = 0%, and 0 rate-limit (429) events.
